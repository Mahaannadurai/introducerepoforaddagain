# Upgraded Chapter Structure: Lists

## 1. Why Lists Matter

Start with real-world motivation.

Most programs work with collections of data.

Examples:

```python
servers = [
    "web-01",
    "web-02",
    "web-03"
]
```

```python
users = [
    "alice",
    "bob",
    "charlie"
]
```

```python
cpu_usages = [
    45,
    67,
    82,
    30
]
```

Without lists:

```python
server1 = "web-01"
server2 = "web-02"
server3 = "web-03"
server4 = "web-04"
server5 = "web-05"
```

This quickly becomes impossible to manage.

Lists allow us to store many related items in a single variable.

# 2. What Is a List?

Visualization:

```text

| web-02   |
+----------+
| web-03   |
+----------+
```

Python remembers:

```text
Index

1 → web-02
2 → web-03
```

# 6. Slicing Lists

One of the most important concepts.

```python
servers = [
    "web-01",
    "web-02",
    "web-03",
    "web-04"
]
```

---

## First Two

```python
servers[:2]
```

Output:

```python
['web-01', 'web-02']
```

Remember:

```text
Start Included
Stop Excluded
```

---

## Last Two

```python
servers[-2:]
```

Output:

```python
['web-03', 'web-04']
```

---

## Step Values

```python
servers[::2]
```

Output:

```python
['web-01', 'web-03']
```

---

## Reverse

```python
servers[::-1]
```

Output:

```python
['web-04', 'web-03', 'web-02', 'web-01']
```

# 7. Lists Are Mutable

This is the most important difference from strings.

Strings:

```python
name = "Alice"

name[0] = "M"
```

Error.

Lists:

```python
users = [
    "alice",
    "bob"
]

users[0] = "alex"

print(users)
```

Output:

```python
['alex', 'bob']
```

Visualization:

```text
Before

0 → alice

After

0 → alex
```

# 8. Adding Items

## append()

Most common list method.

```python

```

Output:

```python

```

---

Multiple appends:

```python

```

## extend()

Very commonly forgotten.


# 16. List Comprehensions

The original chapter introduces them too early.

Move them to the end.

Explain:

Traditional approach:

```python
squares = []

for number in range(5):
    squares.append(number ** 2)
```

Output:

```python
[0, 1, 4, 9, 16]
```

---

List comprehension:

```python
squares = [
    number ** 2
    for number in range(5)
]
```

Output:

```python
[0, 1, 4, 9, 16]
```

---

Visual breakdown:

```text
[number ** 2
 for number in range(5)]
```

Read as:

```text
Create a list of

number²

for every number in range(5)
```

# 18. Debugging Lab

## Error 1

```python
users = ["alice"]

print(users[5])
```

Output:

```text
IndexError:
list index out of range
```

Root cause:

Index doesn't exist.

---

Fix:

```python
print(users[0])
```

---

## Error 2

```python
numbers = [3, 1, 2]

sorted_numbers = numbers.sort()

print(sorted_numbers)
```

Output:

```text
None
```

Why?

`sort()` modifies the list and returns nothing.

Correct:

```python
numbers.sort()

print(numbers)
```

# 19. Common Beginner Mistakes

### Mistake 1

Confusing append and extend.

```python
numbers.append([4,5])
```

Result:

```python
[1,2,3,[4,5]]
```

Not:

```python
[1,2,3,4,5]
```

Use:

```python
numbers.extend([4,5])
```

---

### Mistake 2

Using an invalid index.

```python
users[100]
```

---

### Mistake 3

Expecting sort() to return a list.

```python
new_list = users.sort()
```

Wrong.

# Key Takeaways

- Lists store collections of items.
- Lists are ordered sequences.
- Lists are mutable.
- Indexing starts at 0.
- Negative indexing works from the end.
- Slices exclude the stop position.
- `append()` adds one item.
- `extend()` adds multiple items.
- `pop()` removes by index.
- `remove()` removes by value.
- `sort()` modifies a list in place.
- List comprehensions provide a concise way to build new lists.
- Lists are one of the most frequently used data structures in Python and are essential for automation, DevOps, data processing, and application development.
