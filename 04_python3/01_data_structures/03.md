| \t     | Tab          |
| \      | Backslash    |

# 10. String Immutability

Current chapter only shows the error.

Need deeper explanation.

```python
name = "Alice"

name[0] = "M"
```

Output:

```text
TypeError
```

Why?

Strings are immutable.

Visualization:

```text
"Alice"

Cannot modify character in place.
```

Instead:

```python
name = "M" + name[1:]
```

Output:

```text
Mlice
```

Explain:

New string created.

Old string untouched.

# 17. Common Beginner Mistakes

### Mistake 1

```python
name[0] = "A"
```

Strings are immutable.

---

### Mistake 2

```python
word[1:5]
```

Expecting index 5 included.

It is excluded.

---

### Mistake 3

```python
"Age: " + 25
```

Produces:

```text
TypeError
```

Fix:

```python
f"Age: {25}"
```

# 20. Key Takeaways

- Strings store textual data.
- Strings are sequences.
- Indexing starts at 0.
- Negative indexes work from the end.
- Slices exclude the stop position.
- Strings are immutable.
- Methods create new strings.
- `len()` returns character count.
- f-strings are the preferred formatting technique in Python 3.12+.
- String manipulation is one of the most important programming skills because nearly every application processes text.
