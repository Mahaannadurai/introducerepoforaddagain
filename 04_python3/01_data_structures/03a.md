# Upgraded Chapter Structure: String Formatting (Python 3.12+)

## Why String Formatting Matters

Programs rarely display raw values.

Instead, programs combine text and data.

Examples:

```python
username = "alice"
cpu_usage = 85
server_count = 12
```

We usually want output like:

```text
Hello alice

CPU Usage: 85%

Active Servers: 12
```

String formatting allows Python to inject values into text safely and cleanly.

Real-world uses:

- Log messages
- CLI tools
- DevOps automation
- Monitoring dashboards
- API responses
- Error messages
- Reports
- Configuration generation

Without formatting:

```python
print("CPU Usage:", cpu_usage)
```

With formatting:

```python
print(f"CPU Usage: {cpu_usage}%")
```

Much cleaner.

---

# The Modern Python Rule

For Python 3.12+:

✅ Use f-strings by default

⚠️ Use `.format()` when necessary

📚 Understand `%` formatting because you will encounter it in old codebases

# What Is String Formatting?

String formatting inserts values into strings.

Example:

```python
name = "Alice"

message = f"Hello {name}"

print(message)
```

Output:

```text
Hello Alice
```

Visualization:

```text
Template

"Hello {name}"

        |
        v

Insert value of name

        |
        v

"Hello Alice"
```

---

# Formatted String Literals (f-Strings)

Introduced:

```text
Python 3.6
```

Recommended:

```text
Python 3.12+
```

---

## Basic Example

```python
name = "Alice"

print(f"Hello {name}")
```

Output:

```text
Hello Alice
```

---

## Multiple Variables

```python
name = "Alice"
age = 25

print(f"{name} is {age} years old.")
```

Output:

```text
Alice is 25 years old.
```

---

## Expressions Inside f-Strings

One of the biggest advantages.

```python
a = 10
b = 20

print(f"Total = {a + b}")
```

Output:

```text
Total = 30
```

Python evaluates the expression first.

---

## Method Calls Inside f-Strings

```python
username = "alice"

print(f"User: {username.upper()}")
```

Output:

```text
User: ALICE
```

---

## Real-World DevOps Example

```python
hostname = "web-01"
cpu_usage = 82

print(
    f"Host {hostname} is using "
    f"{cpu_usage}% CPU"
)
```

Output:

```text
Host web-01 is using 82% CPU
```

# Python 3.12 Improvements

Python 3.12 significantly improved f-string parsing.

This now works naturally:

```python
employee = {
    "name": "Alice",
    "age": 30
}

print(
    f"Employee name: {employee["name"]}"
)
```

Output:

```text
Employee name: Alice
```

Older Python versions often required quote juggling.

Python 3.12 makes many f-string expressions easier to write and read.

# Debugging with f-Strings

A hidden superpower.

```python
cpu_usage = 85

print(f"{cpu_usage=}")
```

Output:

```text
cpu_usage=85
```

Multiple values:

```python
host = "web-01"
memory = 2048

print(f"{host=}, {memory=}")
```

Output:

```text
host='web-01', memory=2048
```

Very useful during debugging.

# Formatting Numbers

## Decimal Places

```python
price = 19.9876

print(f"{price:.2f}")
```

Output:

```text
19.99
```

Meaning:

```text
.2

Show 2 digits
after decimal point
```

---

## Width Formatting

```python
price = 19.99

print(f"{price:10.2f}")
```

Output:

```text
     19.99
```

Meaning:

```text
10

Minimum width

2

Decimal places
```

---

## Percentages

```python
usage = 0.87

print(f"{usage:.1%}")
```

Output:

```text
87.0%
```

---

## Thousands Separators

```python
users = 1250000

print(f"{users:,}")
```

Output:

```text
1,250,000
```

Very useful for reports.

---

# Alignment

## Left Align

```python
print(f"{'Alice':<10}")
```

Output:

```text
Alice
```

---

## Center Align

```python
print(f"{'Alice':^10}")
```

Output:

```text
  Alice
```

---

## Right Align

```python
print(f"{'Alice':>10}")
```

Output:

```text
     Alice
```

---

# Building Tables

```python
print(f"{'Server':<10} {'CPU':>5}")
print(f"{'web-01':<10} {85:>5}")
print(f"{'web-02':<10} {63:>5}")
```

Output:

```text
Server       CPU
web-01        85
web-02        63
```

Extremely common in CLI tools.

# String Representation (!r)

Normally:

```python
name = "Alice"

print(f"{name}")
```

Output:

```text
Alice
```

Representation:

```python
print(f"{name!r}")
```

Output:

```text
'Alice'
```

Useful for debugging hidden characters.

# .format() Method

Before f-strings, `.format()` was the preferred solution.

Still common in production code.

---

## Basic Example

```python
name = "Alice"

print(
    "Hello {}".format(name)
)
```

Output:

```text
Hello Alice
```

---

## Multiple Values

```python
print(
    "{} is {} years old".format(
        "Alice",
        25
    )
)
```

Output:

```text
Alice is 25 years old
```

---

## Named Placeholders

```python
print(
    "{user} logged into {server}".format(
        user="alice",
        server="web-01"
    )
)
```

Output:

```text
alice logged into web-01
```

# Common Beginner Mistakes

## Mistake 1

Forgetting the `f`

Wrong:

```python
name = "Alice"

print("Hello {name}")
```

Output:

```text
Hello {name}
```

Correct:

```python
print(f"Hello {name}")
```

---

## Mistake 2

Using `+` unnecessarily

Wrong:

```python
print(
    "Hello " + name +
    ", welcome!"
)
```

Correct:

```python
print(
    f"Hello {name}, welcome!"
)
```

---

## Mistake 3

Mixing Types

Wrong:

```python
age = 25

print("Age: " + age)
```

Error:

```text
TypeError
```

Correct:

```python
print(f"Age: {age}")
```

# Debugging Lab

Broken Code:

```python
name = "Alice"

print("Hello {name}")
```

Output:

```text
Hello {name}
```

Root Cause:

Missing `f`.

Fix:

```python
name = "Alice"

print(f"Hello {name}")
```

Output:

```text
Hello Alice
```

# Key Takeaways

- String formatting inserts values into text.
- f-strings are the preferred approach in Python 3.12+.
- f-strings support variables, expressions, and method calls.
- Numeric formatting controls width, alignment, and precision.
- `!r` displays an object's representation.
- `.format()` is still common in existing codebases.
- `%` formatting is legacy but still worth recognizing.
- Most production Python applications rely heavily on string formatting for logs, reports, automation, APIs, and user-facing output.
