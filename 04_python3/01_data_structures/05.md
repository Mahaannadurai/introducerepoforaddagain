# Upgraded Chapter Structure: Dictionaries

## Why Dictionaries Matter

Lists store data by position.

Example:

```python
servers = [
    "web-01",
    "web-02",
    "web-03"
]
```

To find a server:

```python
servers[1]
```

You must know the position.

But real-world data usually has names.

For example:

```python
employee = {
    "name": "Alice",
    "age": 30,
    "department": "Engineering"
}
```

You don't want:

```python
employee[0]
```

You want:

```python
employee["name"]
```

This is where dictionaries become useful.

---

# What Is a Dictionary?

A dictionary stores data as:

```text
Key → Value
```

Visualization:

```text
"name"       → "Alice"
"age"        → 30
"department" → "Engineering"
```

Instead of storing values by position:

```text
0 → Alice
1 → 30
2 → Engineering
```

Python stores values by key.

---

# Dictionary vs List

List:

```python
user = [
    "Alice",
    30
]
```

Access:

```python
user[0]
```

Output:

```text
Alice
```

Not obvious.

---

Dictionary:

```python
user = {
    "name": "Alice",
    "age": 30
}
```

Access:

```python
user["name"]
```

Output:

```text
Alice
```

Much more readable.

# Dictionary Keys Must Exist

```python
user = {
    "name": "Alice"
}

print(user["email"])
```

Output:

```text
KeyError: 'email'
```

This is one of the most common beginner errors.

---

# Safe Access With get()

Modern Python code often uses `get()`.

```python
user = {
    "name": "Alice"
}

print(user.get("email"))
```

Output:

```text
None
```

No crash.

---

## Default Values

```python
print(
    user.get(
        "email",
        "Not Provided"
    )
)
```

Output:

```text
Not Provided
```

This is extremely useful in production code.

---

# Dictionaries Are Mutable

Unlike strings, dictionaries can be modified.

```python
user = {
    "name": "Alice"
}

user["name"] = "Bob"

print(user)
```

Output:

```python
{'name': 'Bob'}
```

---

# Adding New Keys

# Dictionary Methods

Teach these early because they are used constantly.

---

## keys()

```python
user = {
    "name": "Alice",
    "age": 30
}

print(user.keys())
```

Output:

```text
dict_keys(['name', 'age'])
```

---

## values()

```python
print(user.values())
```

Output:

```text
dict_values(['Alice', 30])
```

---

## items()

```python
print(user.items())
```

Output:

```text
dict_items([
    ('name', 'Alice'),
    ('age', 30)
])
```

Most useful for loops later.

# Dictionary Ordering (Python 3.7+)

Old Python:

```text
Dictionary order not guaranteed
```

Modern Python:

```text
Python 3.7+

Insertion order preserved
```

Example:

```python
data = {
    "a": 1,
    "b": 2,
    "c": 3
}
```

Output remains:

```text
a
b
c
```

in insertion order.

This is now guaranteed behavior.

# Nested Dictionaries

The original chapter introduces nesting but doesn't explain why it matters.

Real-world APIs return nested dictionaries constantly.

---

Example:

```python

    "name": "Alice",
```

```text
employee
```

# Dictionary Merge Operator (Python 3.9+)

Modern Python feature.

```python
defaults = {
    "port": 80
}

custom = {
    "port": 8080,
    "debug": True
}

config = defaults | custom
```

Output:

```python
{
    'port': 8080,
    'debug': True
}
```

Right side wins.

Visualization:

```text
defaults
    port=80

custom
    port=8080

Result
    port=8080
```

# Common Beginner Mistakes

## Mistake 1

Using a missing key.

```python
user["email"]
```

Without checking.

Use:

```python
user.get("email")
```

---

## Mistake 2

Forgetting quotes.

Wrong:

```python
user[name]
```

Python thinks `name` is a variable.

Correct:

```python
user["name"]
```

---

## Mistake 3

Expecting key order in very old Python versions.

Modern Python preserves insertion order.

# Key Takeaways

- Dictionaries store data as key-value pairs.
- Dictionaries are mappings, not sequences.
- Values are accessed using keys.
- Dictionaries are mutable.
- `get()` provides safe key access.
- `keys()`, `values()`, and `items()` are essential methods.
- Dictionaries preserve insertion order in Python 3.7+.
- Nested dictionaries are extremely common in APIs and configuration files.
- The `|` operator merges dictionaries in Python 3.9+.
- Dictionaries are one of the most important data structures in Python and are used extensively in automation, DevOps, web development, APIs, JSON processing, and configuration management.
