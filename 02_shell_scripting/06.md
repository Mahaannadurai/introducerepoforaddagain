# Module 6 — Making Decisions

## What You Will Learn
- How to run different commands based on a condition
- `if`, `else`, `elif` syntax
- How to compare numbers and strings
- How to check if files or directories exist
- Real DevOps decision-making examples

## Why This Matters

A script that runs the same commands regardless of what's happening isn't very smart. Real automation needs to react: "if disk is over 90%, send a warning — otherwise, log that everything is fine." This module is where your scripts start thinking.

## Real Linux Problem

Disk fills up. When it hits 90%, InfraOps wants an alert. When it's under 70%, they want a green light. Between 70–90%, a yellow warning. A script that checks and responds correctly is a basic monitoring tool.

## Simple Explanation

An `if` statement is a fork in the road. Your script reaches the fork, checks a condition, and takes the appropriate path.

```
Disk usage check
        │
        ▼
Is usage > 90%?
    /        \
  Yes          No
   │            │
ALERT        OK message
```

In Bash:

```bash
if [ condition ]; then
    # commands when condition is true
else
    # commands when condition is false
fi
```

`fi` is just `if` backwards — it closes the if block.

## Comparison Operators

**Comparing numbers:**
```bash
[ $A -eq $B ]   # equal to
[ $A -ne $B ]   # not equal to
[ $A -lt $B ]   # less than
[ $A -le $B ]   # less than or equal to
[ $A -gt $B ]   # greater than
[ $A -ge $B ]   # greater than or equal to
```

**Comparing strings:**
```bash
[ "$A" = "$B" ]    # equal
[ "$A" != "$B" ]   # not equal
[ -z "$A" ]        # empty (zero length)
[ -n "$A" ]        # not empty
```

**Checking files and directories:**
```bash
[ -f "$PATH" ]   # is a file (and exists)
[ -d "$PATH" ]   # is a directory (and exists)
[ -e "$PATH" ]   # exists (file or directory)
[ -r "$PATH" ]   # is readable
[ -w "$PATH" ]   # is writable
[ -x "$PATH" ]   # is executable
```

## Guided Example — disk-check.sh

```bash
#!/bin/bash

# disk-check.sh
# Checks disk usage on / and alerts if over threshold
# Part of InfraOps monitoring suite

DISK_PERCENT=$(df / | awk 'NR==2 {print $5}' | tr -d '%')
WARN_THRESHOLD=70
ALERT_THRESHOLD=90

echo "Disk check: ${DISK_PERCENT}% used on /"

if [ "$DISK_PERCENT" -ge "$ALERT_THRESHOLD" ]; then
    echo "ALERT: Disk usage is critical! ${DISK_PERCENT}% used."
    echo "Action required immediately."
elif [ "$DISK_PERCENT" -ge "$WARN_THRESHOLD" ]; then
    echo "WARNING: Disk usage is elevated. ${DISK_PERCENT}% used."
    echo "Monitor closely."
else
    echo "OK: Disk usage is normal. ${DISK_PERCENT}% used."
fi
```

**Line by line:**

- `df / | awk 'NR==2 {print $5}'` — gets the "Use%" column from `df`
- `tr -d '%'` — removes the `%` sign so we have a plain number
- `"$DISK_PERCENT" -ge "$ALERT_THRESHOLD"` — compares numbers (never use `>` for numbers in `[ ]`)
- `elif` — "else if" — adds a second condition before the fallback `else`

## Guided Example — file-check.sh

```bash
#!/bin/bash

# file-check.sh
# Verifies that required config files exist before running

CONFIG_FILE="/etc/hostname"
LOG_DIR="/var/log"
BACKUP_SCRIPT="$HOME/devops-bash-toolkit/scripts/system/system-info.sh"

echo "=== Pre-flight checks ==="

if [ -f "$CONFIG_FILE" ]; then
    echo "OK  Config file exists: $CONFIG_FILE"
else
    echo "FAIL Config file missing: $CONFIG_FILE"
fi

if [ -d "$LOG_DIR" ]; then
    echo "OK  Log directory exists: $LOG_DIR"
else
    echo "FAIL Log directory missing: $LOG_DIR"
fi

if [ -x "$BACKUP_SCRIPT" ]; then
    echo "OK  Backup script is executable"
else
    echo "FAIL Backup script not found or not executable: $BACKUP_SCRIPT"
fi
```

## Common Mistakes

**Using `=` instead of `-eq` for numbers:**
```bash
if [ $DISK -eq 90 ]   # Correct numeric comparison
if [ $DISK = "90" ]   # Works but compares as strings — risky
if [ $DISK == 90 ]    # Works in [[ ]] but not always in [ ]
```

**Forgetting quotes around variables:**
```bash
if [ $NAME = "Alice" ]    # Breaks if NAME is empty or has spaces
if [ "$NAME" = "Alice" ]  # Always quote strings
```

**Wrong spacing inside brackets:**
```bash
if [$DISK -gt 90]; then   # Error: missing spaces around [
if [ $DISK -gt 90 ]; then # Correct: space after [ and before ]
```

**Missing `fi`:**
```bash
if [ ... ]; then
    echo "yes"
# forgot fi — script fails
fi
```

## Quick Recap

- `if [ condition ]; then ... fi` is the basic structure
- `elif` adds more conditions; `else` is the fallback
- Numbers: use `-eq`, `-gt`, `-lt` etc.
- Strings: use `=` and `!=`, always with quotes
- Files: `-f` file, `-d` directory, `-e` exists, `-x` executable
- Always quote variables inside `[ ]`
