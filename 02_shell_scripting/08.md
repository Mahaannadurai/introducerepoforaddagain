# Module 8 ‚Äî Loops

## What You Will Learn
- Why loops exist
- `for` loops ‚Äî repeating over a list
- `while` loops ‚Äî repeating while a condition is true

## Why This Matters

Loops are how you scale. One command on one server is manual work. The same command on 50 servers in a loop is automation. Once you understand loops, scripts stop being tools that help you type ‚Äî they become tools that multiply your effort.

## Real Linux Problem

InfraOps has five log directories that need to be checked every morning. Without loops, you write five separate `ls` commands. With a loop, you write the logic once and let the loop handle all five ‚Äî or fifty.

## Simple Explanation

A `for` loop says: "For each thing in this list, do these steps."

```
List: server1 server2 server3
         ‚îÇ
         ‚ñº
Loop iteration 1: SERVER=server1 ‚Üí echo "Checking server1"
Loop iteration 2: SERVER=server2 ‚Üí echo "Checking server2"
Loop iteration 3: SERVER=server3 ‚Üí echo "Checking server3"
         ‚îÇ
         ‚ñº
Loop ends
```

A `while` loop says: "Keep doing these steps as long as this condition is true."

```
ATTEMPTS=0

Is ATTEMPTS < 3?
    ‚îú‚îÄ‚îÄ YES ‚Üí run command, increment ATTEMPTS, check again
    ‚îî‚îÄ‚îÄ NO ‚Üí exit loop
```

## for Loop Syntax

```bash
# Loop over a list of words
for ITEM in item1 item2 item3; do
    echo "Processing: $ITEM"
done

# Loop over files (glob patterns)
for FILE in /var/log/*.log; do
    echo "Found log: $FILE"
done

# Loop with a number range
for NUMBER in $(seq 1 5); do
    echo "Count: $NUMBER"
done

# C-style for loop (less common in Bash)
for (( i=1; i<=5; i++ )); do
    echo "i=$i"
done
```

## while Loop Syntax

```bash
# While a condition is true
COUNT=1
while [ $COUNT -le 5 ]; do
    echo "Loop $COUNT"
    COUNT=$((COUNT + 1))   # Increment COUNT by 1
done

# Read a file line by line
while IFS= read -r LINE; do
    echo "Line: $LINE"
done < "/path/to/file.txt"

# Infinite loop (use with break to exit)
while true; do
    read -p "Enter q to quit: " INPUT
    if [ "$INPUT" = "q" ]; then
        break
    fi
    echo "You typed: $INPUT"
done
```

`$((expression))` is arithmetic expansion ‚Äî it does math. `$((COUNT + 1))` adds 1 to COUNT.

## Loop Control

```bash
for i in 1 2 3 4 5; do
    if [ $i -eq 3 ]; then
        continue   # Skip this iteration ‚Äî jump to next
    fi
    if [ $i -eq 5 ]; then
        break      # Stop the loop entirely
    fi
    echo $i
done
# Output: 1 2 4
```

## Guided Example ‚Äî multi-server-check.sh

```bash
#!/bin/bash

# multi-server-check.sh
# Checks disk usage on multiple directories
# Simulates checking multiple server mount points

TARGETS=("/var" "/tmp" "/home" "/usr" "/etc")
WARN_THRESHOLD=80

echo "=========================================="
echo "  Multi-Target Disk Check"
echo "  $(date '+%Y-%m-%d %H:%M:%S')"
echo "=========================================="

for TARGET in "${TARGETS[@]}"; do
    # Check if directory exists
    if [ ! -d "$TARGET" ]; then
        echo "SKIP  $TARGET (not found)"
        continue
    fi

    USAGE=$(df "$TARGET" | awk 'NR==2 {print $5}' | tr -d '%')

    if [ "$USAGE" -ge "$WARN_THRESHOLD" ]; then
        echo "WARN  $TARGET ‚Äî ${USAGE}% used"
    else
        echo "OK    $TARGET ‚Äî ${USAGE}% used"
    fi
done

echo "=========================================="
echo "Check complete."
```

**New syntax explained:**
- `TARGETS=(...)` ‚Äî an array (a variable holding multiple values)
- `"${TARGETS[@]}"` ‚Äî expands all elements of the array
- `! -d` ‚Äî the `!` negates: "is NOT a directory"

## Common Mistakes

**Forgetting `done`:**
```bash
for i in 1 2 3; do
    echo $i
# Missing done ‚Äî syntax error
done
```

**Wrong array expansion:**
```bash
SERVERS=("web01" "web02" "web03")
for S in $SERVERS; do    # WRONG ‚Äî only iterates first element
for S in "${SERVERS[@]}"; do  # CORRECT
```

**Infinite loop without an exit condition:**
```bash
while true; do
    echo "looping"
    # No break ‚Äî runs forever
done
```

Always have a `break`, a condition that becomes false, or a `Ctrl+C` safety valve when testing.

## Quick Recap

- `for ITEM in list; do ... done` ‚Äî iterate over items
- `while [ condition ]; do ... done` ‚Äî repeat while condition is true
- `break` ‚Äî exit the loop
- `continue` ‚Äî skip to next iteration
- Arrays use `()` and expand with `"${ARRAY[@]}"`
- Math uses `$((expression))`

## Interview Corner ‚Äî Loops

**Why interviewers ask:** Loops test whether you can automate repetitive tasks ‚Äî the core job of any DevOps engineer. They'll give you a scenario ("how would you check if all 20 servers are responding?") and expect you to reach for a loop.

**What they expect:**
- You can write both `for` and `while` loops
- You understand the difference and when to use each
- You can loop over a file, a list, and a range
- You know how to break out of a loop safely

**Common beginner mistakes:**
- Using `$ARRAY` instead of `"${ARRAY[@]}"`
- Infinite loops in while with no exit condition
- Forgetting to quote variables inside loops (especially filenames with spaces)

**Practice question:** "Write a loop that checks if each server in a file called `servers.txt` is reachable with ping."

```bash
while IFS= read -r SERVER; do
    if ping -c 1 -W 1 "$SERVER" &>/dev/null; then
        echo "UP  : $SERVER"
    else
        echo "DOWN: $SERVER"
    fi
done < servers.txt
```
