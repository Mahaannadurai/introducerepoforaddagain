## Why This Matters

Every Linux system has a shell. Every DevOps job uses it. When a server breaks at 2am, the shell is how engineers fix it — fast, without a graphical interface. When you need to automate a thousand tasks, the shell is the tool. Bash is the most widely used shell in the world.

## Real Linux Problem

You start work at InfraOps. Your manager drops this task on your desk:

> "Every morning, check which servers are running out of disk space and email me the list."

There are 50 servers. You'd have to log into each one, type a command, check the result, write it down, and repeat. That's two hours of boring, error-prone work — every single day.

A Bash script can do all 50 servers in seconds while you drink your coffee.

## Simple Explanation

Think of Linux like a city. The **kernel** is the power plant — it controls the hardware. The **shell** is the front desk — it takes your requests and passes them to the power plant.

Bash is just one type of shell. Its full name is **Bourne Again SHell**. It's been the default shell on Linux and macOS for decades.

When you type `ls` or `cd` into your terminal, Bash is the program reading those words, understanding them, and telling Linux what to do.

```
You type a command
        ↓
   Bash reads it
        ↓
Linux kernel runs it
        ↓
   Result appears
```

A **script** is just a text file full of commands. Instead of typing one command at a time, you write many commands in a file and run them all at once — in order, automatically.

## Guided Example

Open your WSL Ubuntu terminal. Type the following commands one by one:

```bash
# What shell are you running?
echo $SHELL

# What version of Bash?
bash --version

# Where does Bash live?
which bash
```

You should see something like:
```
/bin/bash
GNU bash, version 5.1.16(1)-release
/usr/bin/bash
```

This confirms Bash 5 is installed and ready.

## Hands-On Lab 2 — Set Up the Project

**Objective:** Create the `devops-bash-toolkit` folder structure.

```bash
# Create the main project folder
mkdir ~/devops-bash-toolkit

# Move into it
cd ~/devops-bash-toolkit

# Create all subdirectories at once
mkdir -p scripts/hello scripts/system scripts/files scripts/monitoring scripts/users
mkdir lib config logs

# Verify the structure
ls -R
```

## Common Mistakes

| Mistake | What Happens | Fix |
|---|---|---|
| Typing `Bash` with capital B | `command not found` | Always lowercase: `bash` |
| Forgetting to `cd` first | Files created in wrong folder | Always `pwd` to check location |
| Using Windows line endings | Script fails silently | Use VS Code with LF line endings |
