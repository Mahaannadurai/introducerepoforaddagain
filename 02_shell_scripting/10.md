# Module 10 — Working With Files

## What You Will Learn
- How to read files line by line
- How to write and overwrite a file
- How to append to a file without erasing it
- How to check file properties
- Building a file report script

## Why This Matters

Servers generate files constantly: log files, config files, data files. Scripts that can't read and write files are limited. This module gives you the ability to process real data.

## Key Concepts

**Overwrite vs. Append:**
```bash
echo "new content" > file.txt    # Overwrites — erases existing content
echo "more content" >> file.txt  # Appends — adds to end of file
```

This is the most important distinction in file I/O. Using `>` when you meant `>>` is a common way to accidentally destroy data.

**Reading a file line by line:**
```bash
while IFS= read -r LINE; do
    echo "Line: $LINE"
done < "filename.txt"
```

`IFS=` (empty) preserves leading whitespace. `-r` prevents backslash interpretation. These two options together give you the line exactly as it appears in the file — no surprises.

## Guided Example — file-report.sh

```bash
#!/bin/bash

# file-report.sh
# Generates a report about a specified directory's files
# Usage: ./file-report.sh [directory]

source ~/devops-bash-toolkit/lib/utils.sh

TARGET_DIR="${1:-$HOME}"
REPORT_FILE="$HOME/devops-bash-toolkit/logs/file-report-$(date +%Y%m%d).txt"

print_header "File Report: $TARGET_DIR"

# Check directory exists
if [ ! -d "$TARGET_DIR" ]; then
    echo "Error: Directory not found: $TARGET_DIR"
    exit 1
fi

{
    echo "File Report"
    echo "Generated : $(get_timestamp)"
    echo "Directory : $TARGET_DIR"
    echo ""

    echo "=== File count ==="
    find "$TARGET_DIR" -maxdepth 1 -type f | wc -l

    echo ""
    echo "=== Files ==="
    for FILE in "$TARGET_DIR"/*; do
        if [ -f "$FILE" ]; then
            SIZE=$(du -sh "$FILE" 2>/dev/null | cut -f1)
            echo "$SIZE  $(basename "$FILE")"
        fi
    done

} | tee "$REPORT_FILE"

echo ""
echo "Report saved to: $REPORT_FILE"
```

**New concepts:**
- `{ ... } | tee file` — group commands in `{}` and both print to screen AND write to file simultaneously
- `tee` — reads from stdin, writes to both stdout and a file
- `find "$DIR" -maxdepth 1 -type f` — lists only files (not subdirectories) directly in the directory
- `du -sh` — disk usage, human-readable, summary
- `basename` — strips the directory path, leaving just the filename

## Common Mistakes

**Using `>` instead of `>>` when you want to append:**
```bash
echo "entry 1" > logfile.txt   # Creates/overwrites
echo "entry 2" > logfile.txt   # Overwrites again — entry 1 is gone!
echo "entry 2" >> logfile.txt  # Appends — both entries exist
```

**Not quoting file paths with spaces:**
```bash
FILE="my report.txt"
cat $FILE          # Breaks — Bash sees: cat my report.txt (two arguments)
cat "$FILE"        # Correct
```

## Quick Recap

- `>` overwrites a file, `>>` appends
- `while IFS= read -r LINE; do ... done < file` reads line by line
- Always quote file path variables
- `tee` writes to file and screen simultaneously
