# Module 11 — Working With Directories

## What You Will Learn
- How to check if a directory exists
- How to create directories (including nested ones)
- How to loop over directory contents
- How to clean up old directories

## Guided Example — backup-folder.sh

```bash
#!/bin/bash

# backup-folder.sh
# Creates a timestamped backup of a directory

source ~/devops-bash-toolkit/lib/utils.sh

SOURCE_DIR="${1:-$HOME/devops-bash-toolkit/config}"
BACKUP_ROOT="$HOME/devops-bash-toolkit/logs/backups"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="$BACKUP_ROOT/backup_$TIMESTAMP"

print_header "Folder Backup"

# Validate source
if [ ! -d "$SOURCE_DIR" ]; then
    echo "ERROR: Source directory does not exist: $SOURCE_DIR"
    exit 1
fi

# Create backup root if it doesn't exist
if [ ! -d "$BACKUP_ROOT" ]; then
    mkdir -p "$BACKUP_ROOT"
    echo "Created backup root: $BACKUP_ROOT"
fi

# Create the timestamped backup
cp -r "$SOURCE_DIR" "$BACKUP_DIR"

if [ $? -eq 0 ]; then
    print_status "OK" "Backup created: $BACKUP_DIR"
    FILE_COUNT=$(find "$BACKUP_DIR" -type f | wc -l)
    echo "Files backed up: $FILE_COUNT"
else
    print_status "FAIL" "Backup failed"
    exit 1
fi
```

`$?` is a special variable — it holds the exit code of the last command. `0` = success. Anything else = failure. This pattern (`if [ $? -eq 0 ]`) is how you check whether the previous command worked. We cover this fully in Module 13.

## Hands-On Lab

## Quick Recap

- `mkdir -p` creates a directory and all parent directories
- `cp -r` copies directories recursively
- `$?` checks the exit code of the previous command
