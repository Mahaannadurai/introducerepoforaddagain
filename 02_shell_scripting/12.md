# Module 12 — Logging

## What You Will Learn
- Why logging is essential in automation scripts
- How to write timestamped log entries
- How to set log levels (INFO, WARN, ERROR)
- Building a shared logging library

## Why This Matters

When a script runs at 3am and something goes wrong, the only way to find out what happened is the log file. Without logging, troubleshooting automated scripts is nearly impossible.

## Guided Example — lib/logging.sh

```bash
#!/bin/bash

# lib/logging.sh
# Shared logging functions for the InfraOps toolkit
# Source this file: source lib/logging.sh

LOG_FILE="${LOG_FILE:-$HOME/devops-bash-toolkit/logs/toolkit.log}"
LOG_LEVEL="${LOG_LEVEL:-INFO}"

# Ensure log directory exists
mkdir -p "$(dirname "$LOG_FILE")"

# Internal: write a log entry
_write_log() {
    local LEVEL="$1"
    local MESSAGE="$2"
    local TIMESTAMP
    TIMESTAMP=$(date "+%Y-%m-%d %H:%M:%S")
    local ENTRY="[$TIMESTAMP] [$LEVEL] $MESSAGE"

    # Always write to file
    echo "$ENTRY" >> "$LOG_FILE"

    # Print to screen based on level
    case "$LEVEL" in
        ERROR) echo "$ENTRY" >&2 ;;  # Errors go to stderr
        WARN)  echo "$ENTRY" ;;
        INFO)  echo "$ENTRY" ;;
        DEBUG) [ "$LOG_LEVEL" = "DEBUG" ] && echo "$ENTRY" ;;
    esac
}

log_info()  { _write_log "INFO"  "$1"; }
log_warn()  { _write_log "WARN"  "$1"; }
log_error() { _write_log "ERROR" "$1"; }
log_debug() { _write_log "DEBUG" "$1"; }
```

**Using it:**

```bash
#!/bin/bash
source ~/devops-bash-toolkit/lib/utils.sh
source ~/devops-bash-toolkit/lib/logging.sh

log_info "Script started"
log_info "Checking disk usage"

USAGE=$(df / | awk 'NR==2 {print $5}' | tr -d '%')

if [ "$USAGE" -gt 90 ]; then
    log_error "Disk critical: ${USAGE}%"
else
    log_info "Disk OK: ${USAGE}%"
fi

log_info "Script complete"
```

**Log output:**
```
[2026-06-09 14:32:01] [INFO] Script started
[2026-06-09 14:32:01] [INFO] Checking disk usage
[2026-06-09 14:32:01] [INFO] Disk OK: 42%
[2026-06-09 14:32:01] [INFO] Script complete
```

## Quick Recap

- Always log with timestamps
- Separate log levels: DEBUG, INFO, WARN, ERROR
- Write errors to stderr with `>&2`
- Shared logging library means one change improves all scripts
