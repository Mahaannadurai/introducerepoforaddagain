
# Module 14 — Useful Linux Commands in Scripts

## What You Will Learn
- `grep` — search text for patterns
- `wc` — count lines, words, characters
- `cut` — extract columns from structured text
- `sort` — sort lines alphabetically or numerically
- `uniq` — deduplicate sorted lines
- How to combine these with pipes in scripts

## Why This Matters

Raw command output is often more than you need. These tools let you extract exactly the data your script needs — the third column of a log file, the unique IP addresses in an access log, the top 10 most frequent errors. Together they form the foundation of text processing in Bash.

## Quick Reference

```bash
# grep: find lines matching a pattern
grep "ERROR" app.log                   # Lines containing ERROR
grep -i "error" app.log                # Case-insensitive
grep -c "ERROR" app.log                # Count of matching lines
grep -v "DEBUG" app.log                # Lines NOT containing DEBUG

# wc: count things
wc -l file.txt     # Count lines
wc -w file.txt     # Count words
wc -c file.txt     # Count bytes

# cut: extract columns (delimiter-separated)
echo "alice:1001:admin" | cut -d: -f1   # alice (field 1)
echo "alice:1001:admin" | cut -d: -f3   # admin (field 3)
cut -d: -f1 /etc/passwd                  # All usernames

# sort: sort lines
sort names.txt             # Alphabetical
sort -r names.txt          # Reverse alphabetical
sort -n numbers.txt        # Numeric sort
sort -k2 data.txt          # Sort by second field

# uniq: remove adjacent duplicate lines (use after sort)
sort names.txt | uniq              # Unique names
sort access.log | uniq -c          # Count of each unique line
sort access.log | uniq -c | sort -rn | head -10  # Top 10 most frequent
```

## Guided Example — log-analyzer.sh

```bash
#!/bin/bash

# log-analyzer.sh
# Analyzes a log file and produces a summary report

source ~/devops-bash-toolkit/lib/utils.sh
source ~/devops-bash-toolkit/lib/logging.sh

LOG_PATH="${1:-/var/log/auth.log}"

if [ ! -f "$LOG_PATH" ] && [ ! -r "$LOG_PATH" ]; then
    log_error "Cannot read log file: $LOG_PATH"
    exit 1
fi

print_header "Log Analysis: $(basename "$LOG_PATH")"

TOTAL_LINES=$(wc -l < "$LOG_PATH")
ERROR_COUNT=$(grep -ci "error\|fail\|denied" "$LOG_PATH" 2>/dev/null || echo 0)
WARN_COUNT=$(grep -ci "warn" "$LOG_PATH" 2>/dev/null || echo 0)

echo "Total lines    : $TOTAL_LINES"
echo "Errors/failures: $ERROR_COUNT"
echo "Warnings       : $WARN_COUNT"

echo ""
echo "=== Most recent errors ==="
grep -i "error\|fail\|denied" "$LOG_PATH" 2>/dev/null | tail -5

log_info "Log analysis complete for $LOG_PATH"
```

## Quick Recap

- `grep` searches; `-i` case-insensitive, `-v` invert, `-c` count
- `wc -l` counts lines
- `cut -d: -f2` extracts column 2 using `:` as delimiter
- `sort | uniq -c | sort -rn` is the pattern for frequency counts
