# Module 5 — Working With Command Output

## What You Will Learn
- How to capture the output of any command into a variable
- The `$(command)` syntax
- How to use captured output in strings and conditions
- Real DevOps examples of command substitution

## Why This Matters

Your server generates data constantly — disk usage, logged-in users, running processes, dates. Command substitution lets you grab any of that data and use it in your scripts. This is how monitoring scripts are built.

## Real Linux Problem

You want to send a log entry every time your backup script runs, showing the exact timestamp. Without command substitution, the timestamp is hardcoded. With it, every run captures the live time automatically.

## Simple Explanation

`$(command)` means: "run this command right now, and use its output as if you typed it directly."

```bash
# Without command substitution:
echo "Today is DATE"       # Prints: Today is DATE (useless)

# With command substitution:
echo "Today is $(date)"    # Prints: Today is Tue Jun  9 2026

# Saving to a variable:
TODAY=$(date)
echo "Today is $TODAY"     # Same result, but reusable
```

Think of it like a vending machine. `$(date)` is the slot where you insert the request — the result falls out and replaces the slot.

## Common Commands to Capture

```bash
# Current date and time
NOW=$(date)
TODAY=$(date +%Y-%m-%d)       # Format: 2026-06-09
TIME=$(date +%H:%M:%S)        # Format: 14:32:01

# Who is logged in
CURRENT_USER=$(whoami)
LOGGED_IN=$(who | wc -l)      # Count of logged-in users

# System information
HOSTNAME=$(hostname)
KERNEL=$(uname -r)
UPTIME=$(uptime -p)

# Disk usage
DISK_USED=$(df -h / | awk 'NR==2 {print $5}')   # e.g. "42%"

# Memory
TOTAL_MEM=$(free -m | awk 'NR==2 {print $2}')   # in MB
FREE_MEM=$(free -m | awk 'NR==2 {print $4}')

# Number of processes
PROCESS_COUNT=$(ps aux | wc -l)
```

Don't worry about `awk` yet — it's introduced in Module 14. Just know that combining commands with `|` (pipe) is already something you know, and `$()` can capture the result of a pipe chain too.

## Guided Example — server-snapshot.sh

```bash
#!/bin/bash

# server-snapshot.sh
# Takes a point-in-time snapshot of key server metrics
# Part of the InfraOps monitoring suite

# Capture all the values we need
SNAPSHOT_TIME=$(date "+%Y-%m-%d %H:%M:%S")
HOSTNAME=$(hostname)
CURRENT_USER=$(whoami)
KERNEL=$(uname -r)
UPTIME=$(uptime -p)
DISK_USAGE=$(df -h / | awk 'NR==2 {print $5}')
LOGGED_USERS=$(who | wc -l)
LOAD_AVG=$(uptime | awk -F'load average:' '{print $2}' | xargs)

echo "============================================"
echo "  Server Snapshot — $SNAPSHOT_TIME"
echo "============================================"
echo "Hostname      : $HOSTNAME"
echo "Current user  : $CURRENT_USER"
echo "Kernel        : $KERNEL"
echo "Uptime        : $UPTIME"
echo "Disk usage /  : $DISK_USAGE"
echo "Logged users  : $LOGGED_USERS"
echo "Load average  : $LOAD_AVG"
echo "============================================"
```

```bash
chmod +x ~/devops-bash-toolkit/scripts/system/server-snapshot.sh
./server-snapshot.sh
```

## Common Mistakes

**Mistake 1: Using backticks instead of `$()`**
```bash
NOW=`date`        # Old style — works but harder to read and nest
NOW=$(date)       # Modern style — preferred
```

**Mistake 2: Spaces inside `$()`**
```bash
NOW=$( date )     # Works but unusual
NOW=$(date)       # Standard
```

**Mistake 3: Forgetting quotes when the output has spaces**
```bash
MSG=$(echo "hello world")
if [ $MSG = "hello world" ]; then  # BREAKS — $MSG expands to two words
if [ "$MSG" = "hello world" ]; then  # Correct — always quote
```

## Quick Recap

- `$(command)` runs a command and replaces itself with the output
- Store the result in a variable: `VAR=$(command)`
- You can pipe inside `$()`: `VAR=$(command | other-command)`
- Always quote variables that come from command output when used in comparisons
